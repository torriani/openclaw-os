OpenClaw OS

16 skills instaladas ~/.claude/skills/openclaw-os* pipeline 7 fases · ~90min

Skill do Claude Code que cria e gerencia agentes AI pessoais (claws) rodando 24/7 no VPS dos seus alunos. Pipeline guiado de 7 fases, do zero ao Telegram funcionando.

TL;DR — só me dê o comando

Pra criar um claw novo do zero, abre o Claude Code e digita:

/openclaw-os:start

Pronto. A skill assume, faz as 7 perguntas/checks que precisa, e te conduz até o claw rodando no Telegram. Tempo total: ~90 minutos se for primeira vez.

Resposta direta à sua dúvida

Sim — /openclaw-os:start chama as 7 fases automaticamente em sequência. Você não precisa decorar mais nenhum comando. Só esse.

Como a skill funciona

A skill funciona como uma recepcionista que te guia por um prédio. Cada andar do prédio é uma fase. Você nunca precisa saber qual elevador apertar — a recepcionista te leva.

Fluxo visual

VOCÊ DIGITA: /openclaw-os:start │ ▼ ┌───────────────────────────┐ │ Skill principal (start) │ ← orquestrador └───────────────────────────┘ │ │ valida pré-requisitos (VPS, ChatGPT, Telegram) │ pergunta nome do claw │ cria state.json │ ▼ PHASE 1 (memória) → "rode /openclaw-os:phase-1" │ │ você roda, ela executa, salva estado │ você volta e digita "ok" ▼ PHASE 2 (credenciais) → "rode /openclaw-os:phase-2" │ ▼ PHASE 3 (infra + sec) → ... │ ▼ ... segue até PHASE 7 ... │ ▼ GO-LIVE ✓ claw no Telegram

Por que não roda tudo automático?

Cada fase pede confirmação humana porque algumas tarefas dependem do aluno (copiar prompt no ChatGPT, criar bot no @BotFather, colar IP do VPS). Sem o aluno, a skill não consegue avançar sozinha. O checkpoint é onde você verifica se a fase deu certo antes de seguir.

Pré-requisitos do aluno

Antes do aluno digitar /openclaw-os:start, ele precisa ter os 3 itens abaixo. Sem eles, a skill para na primeira tela.

Atenção

Esta versão usa ChatGPT (OpenAI), não Anthropic. O aluno precisa de chave da OpenAI (sk-...), não da Anthropic.

Fluxo padrão (caminho feliz)

Esse é o roteiro completo do que acontece numa execução normal, do início ao fim.

Aluno abre Claude Code e digita

/openclaw-os:start

A skill carrega e mostra a tela de boas-vindas + checklist dos 3 pré-requisitos.

Aluno confirma os 3 pré-requisitos

Skill pergunta um por um. Se faltar algum, ela para e orienta como conseguir.

Aluno escolhe nome do claw

Ex: "Alfred", "Sofia", "Zeus". A skill vira em slug: alfred.

Skill cria state.json e mostra roadmap

~/.openclaw/alfred/state.json   # memória do pipeline
~/.openclaw/alfred/.env          # credenciais (criado em phase-2)

Mostra as 7 fases em ordem com tempo estimado.

Skill diz: "rode /openclaw-os:phase-1"

Aluno digita /openclaw-os:phase-1 no prompt. Carrega a skill da Phase 1, executa, atualiza state.json.

Aluno volta ao :start e diz "ok"

Skill principal lê o state.json, vê que phase-1 está em phases_completed, libera phase-2.

Repete passos 5–6 para phases 2, 3, 4, 5, 6, 7

A cada fase: skill pede o comando, aluno roda, volta, diz "ok", próxima fase libera.

Após Phase 7: GO-LIVE

Skill marca go_live: true, mostra resumo final, pede pro aluno mandar "oi" no Telegram pra testar.

Regra de ouro

90% do tempo você só usa /openclaw-os:start. Os outros comandos são pra situações específicas.

As 7 fases em detalhe

#FaseO que entregaTempo
1Memory ExtractionUSER.md com perfil completo do aluno (extraído via ChatGPT)10min
2Credentials.env com IP do VPS, ChatGPT API key, Telegram bot token + chat_id10min
3Infra + Security 9-layerOpenClaw rodando + UFW, Fail2ban, SSH key-only, dmPolicy, audit log, systemd hardening20min
4Identity + Memory 4-layerSOUL.md, IDENTITY.md, AGENTS.md, BOOT.md + memória working/episodic/semantic/procedural15min
5Skills + Crons1+ skill instalada + cron heartbeat 5min + 1 cron proativo15min
6Costs + ImmuneModel split (gpt-4o-mini default), rate limit, budget alert, watchdog, OOM guard, backup diário, audit semanal10min
7Mission ControlHealth check 9 itens + smoke test Telegram + go_live=true10min

Por que essa ordem importa

Cada fase é gate da próxima. Não dá pra pular:

  • Phase 2 (credenciais) precisa do USER.md de Phase 1 pra customizar
  • Phase 3 (infra) precisa das credenciais de Phase 2 pra logar no VPS
  • Phase 4 (identity) precisa da infra de Phase 3 pra rodar OpenClaw
  • Phase 5 (skills) precisa da identidade de Phase 4 pra fazer sentido
  • Phase 6 (immune) precisa de Phase 5 pra ter o que monitorar
  • Phase 7 (mission control) valida que tudo o resto passou

Troubleshooting

SintomaCausa provávelSolução
SSH não conecta no VPSIP errado, senha errada, ou VPS não está em Ubuntu 24.04Confere no painel Hostinger. Re-roda phase-2.
ChatGPT API retorna 401Chave inválida ou sem créditosplatform.openai.com/api-keys → nova chave + crédito
Telegram bot não respondeToken errado ou aluno não mandou "oi" pro botRe-roda phase-2 step Telegram
Skill :start não apareceClaude Code precisa reiniciar pra ver skills novasFecha e abre Claude Code
state.json corrompidoEdição manual quebrou JSONApaga ~/.openclaw/{claw}/state.json, recomeça com :start
OpenClaw service crashouOOM, dependência quebrada, ou config errada:status mostra logs. Watchdog reinicia em 60s.

FAQ

Eu preciso decorar todos os comandos?

Não. Só /openclaw-os:start. Os outros aparecem no autocomplete quando você digita /openclaw-os:.

Posso rodar pra 5 alunos ao mesmo tempo?

Sim. Cada aluno roda /openclaw-os:start na sessão dele. Cada claw tem seu próprio diretório em ~/.openclaw/{slug}/, isolado.

E se eu fechar o terminal no meio?

Tudo bem. Próxima vez é só /openclaw-os:resume. A skill lê o state.json e retoma da última fase salva.

Quanto custa por mês?

VPS Hostinger ~$5–10 + ChatGPT API ~$5–15 (com model split) = $10–25/mês por claw.

Os squads originais ainda existem?

Sim, em squads/openclaw-creator/ e squads/openclaw-manager/. Não foram apagados — a skill OpenClaw OS é a versão fundida e usa templates dos dois.

Como aluno mostra o claw funcionando?

Roda /openclaw-os:status {claw} — dashboard com 10 checks verdes + uptime + gastos do mês.

Posso modificar a skill?

Pode. Os arquivos estão em ~/.claude/skills/openclaw-os-*. Edita o SKILL.md de cada um. Reinicia Claude Code pra recarregar.

Como distribuir pros alunos?

Eles precisam ter as 16 skills no ~/.claude/skills/ deles. Você pode:

  • Manual: zipa ~/.claude/skills/openclaw-os* e manda
  • Via skill-installer: empacota como skill instalável
  • Via repo: aluno clona repo seu e copia pra ~/.claude/skills/
Onde tirar dúvida

Roda /openclaw-os:help dentro do Claude Code pra ajuda contextual. Se a skill travar, abre uma sessão nova e roda /openclaw-os:resume.
